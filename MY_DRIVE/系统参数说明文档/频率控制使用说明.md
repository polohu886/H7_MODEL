# 串口屏频率控制使用指南

## 概述

本模块实现了通过串口屏控制AD9959 DDS芯片输出频率的功能。串口屏通过UART2发送特定格式的数据包，单片机解析后自动更新AD9959的输出频率。

## 数据包格式

### 帧结构
```
55 01 00 00 03 E8 EC FF
├─ 55       帧头（固定0x55）
├─ 01       命令符（0x01 = 设置频率）
├─ 00 00 03 E8   频率数据（4字节，大端序）
├─ EC       校验码（前面所有数据字节的累加和）
└─ FF       帧尾（固定0xFF）
```

### 参数说明

| 位置 | 字段 | 长度 | 说明 | 范围 |
|------|------|------|------|------|
| 0 | 帧头 | 1字节 | 固定值0x55 | 0x55 |
| 1 | 命令符 | 1字节 | 0x01表示设置频率 | 0x01 |
| 2-5 | 频率数据 | 4字节 | 大端序整数，单位Hz | 0~4294967295 |
| 6 | 校验码 | 1字节 | 字节[1]~[5]的累加和 | - |
| 7 | 帧尾 | 1字节 | 固定值0xFF | 0xFF |

### 示例

**设置频率为1000Hz：**
```
数据包：55 01 00 00 03 E8 EC FF
解析：
  - 帧头：0x55
  - 命令：0x01（设置频率）
  - 频率：0x000003E8 = 1000（十进制）
  - 校验：0x01 + 0x00 + 0x00 + 0x03 + 0xE8 = 0xEC
  - 帧尾：0xFF
```

**设置频率为50000Hz：**
```
数据包：55 01 00 00 C3 50 D9 FF
解析：
  - 频率：0x0000C350 = 50000（十进制）
  - 校验：0x01 + 0x00 + 0x00 + 0xC3 + 0x50 = 0xD9
```

## 工作原理

### 系统流程

```
串口屏发送数据
    ↓
UART2接收（中断）→ 环形缓冲区
    ↓
FrequencyControl_Process()
    ↓
PACK_ParseFromRingBuffer() ← 查找帧头/帧尾
    ↓
PACK_ParseFrame() ← 校验和检查，解析数据
    ↓
gFrequencyCmd填充 ← 自动填充全局结构体
    ↓
命令检查 & Write_Frequence()
    ↓
AD9959输出频率更新
```

### 关键函数调用链

```c
// 初始化（仅调用一次）
FrequencyControl_Init()
  └─ PACK_SetTemplate()  // 设置数据包解析模板

// 主循环（周期调用）
FrequencyControl_Process()
  └─ PACK_ParseFromRingBuffer()
      ├─ UART2_GetRxBufferLength()    // 获取接收数据长度
      ├─ UART2_GetReceivedData()      // 读取数据
      └─ PACK_ParseFrame()             // 验证并解析
          └─ 自动填充 gFrequencyCmd
  └─ Write_Frequence(0, frequency)    // 更新AD9959
```

## 代码实现

### 1. frequency_control.h 头文件

定义了模块的公共接口和数据结构：

```c
// 命令符定义
#define CMD_SET_FREQUENCY    0x01

// 数据包结构
typedef struct {
    uint8_t cmd;        // 命令符
    uint32_t frequency; // 频率值（Hz）
} FrequencyCommand_t;

// 导出函数
void FrequencyControl_Init(void);
void FrequencyControl_Process(void);
void FrequencyControl_SetFrequency(uint32_t frequency);
```

### 2. frequency_control.c 实现文件

核心实现逻辑：

**模块初始化：**
```c
void FrequencyControl_Init(void)
{
    PackVariable_t vars[2];
    
    // 注册命令符变量
    vars[0].type = PACK_TYPE_BYTE;
    vars[0].variable = (void *)&gFrequencyCmd.cmd;
    
    // 注册频率变量（4字节大端序）
    vars[1].type = PACK_TYPE_INT;
    vars[1].variable = (void *)&gFrequencyCmd.frequency;
    
    PACK_SetTemplate(vars, 2);
}
```

**命令处理：**
```c
void FrequencyControl_Process(void)
{
    // 解析UART2接收的数据包
    PACK_ParseFromRingBuffer();
    
    // 如果接收到有效命令，更新AD9959
    if (gFrequencyCmd.cmd == CMD_SET_FREQUENCY && gFrequencyCmd.frequency > 0)
    {
        Write_Frequence(0, gFrequencyCmd.frequency);
        Write_Amplitude(0, 500);
        Write_Phase(0, 0);
        AD9959_IO_Update();
        
        // 清除命令，防止重复执行
        gFrequencyCmd.cmd = 0;
        gFrequencyCmd.frequency = 0;
    }
}
```

### 3. main.c 集成

**初始化阶段：**
```c
/* Initialize MAX262 and program three logical channels with same fc/Q */
MAX262_Init();
Init_AD9959();   
Write_Frequence(0,50000);
Write_Amplitude(0, 500);
Write_Phase(0, 0);
AD9959_IO_Update();

/* Initialize frequency control module to handle UART commands */
FrequencyControl_Init();
```

**主循环：**
```c
while (1)
{
    /* Process frequency control commands from UART */
    FrequencyControl_Process();
}
```

## 校验码计算

校验码为帧头后所有数据字节的累加和（8位）：

```
校验码 = (cmd + freq_byte0 + freq_byte1 + freq_byte2 + freq_byte3) & 0xFF
```

### Python计算示例

```python
def calc_checksum(cmd, frequency):
    """计算数据包校验码"""
    # 频率转换为4字节大端序
    freq_bytes = frequency.to_bytes(4, byteorder='big')
    # 累加
    checksum = cmd + sum(freq_bytes)
    return checksum & 0xFF

# 示例：设置1000Hz
cmd = 0x01
freq = 1000
checksum = calc_checksum(cmd, freq)
print(f"校验码：0x{checksum:02X}")  # 输出：0xEC
```

## 故障排查

### 问题1：频率不更新

1. **检查UART2配置**
   - 验证UART2是否正确初始化
   - 确认波特率设置正确

2. **检查数据包格式**
   - 使用串口监视器验证发送的数据
   - 确保帧头为0x55，帧尾为0xFF
   - 验证校验码计算是否正确

3. **检查程序流程**
   ```c
   // 在 FrequencyControl_Process() 中添加调试输出
   if (gFrequencyCmd.cmd == CMD_SET_FREQUENCY)
   {
       printf("收到频率命令: %u Hz\n", gFrequencyCmd.frequency);
   }
   ```

### 问题2：校验码错误

检查校验码计算：
```c
uint8_t checksum = cmd_byte + freq_byte[0] + freq_byte[1] + freq_byte[2] + freq_byte[3];
```

### 问题3：AD9959不响应

1. 验证 `Write_Frequence()` 函数是否正常工作
2. 检查 `AD9959_IO_Update()` 是否调用
3. 确认AD9959的SPI通信正常

## 扩展功能

### 添加更多命令

可以通过扩展命令符来添加新功能：

```c
#define CMD_SET_FREQUENCY    0x01  // 既有：设置频率
#define CMD_SET_AMPLITUDE    0x02  // 新增：设置幅度
#define CMD_SET_PHASE        0x03  // 新增：设置相位
```

修改 `frequency_control.c` 的数据结构：

```c
typedef struct {
    uint8_t cmd;         // 命令符
    uint32_t frequency;  // 频率或其他参数
    uint16_t amplitude;  // 幅度（可选）
    uint16_t phase;      // 相位（可选）
} FrequencyCommand_t;
```

在 `FrequencyControl_Process()` 中添加分支处理：

```c
if (gFrequencyCmd.cmd == CMD_SET_FREQUENCY)
{
    Write_Frequence(0, gFrequencyCmd.frequency);
    // ...
}
else if (gFrequencyCmd.cmd == CMD_SET_AMPLITUDE)
{
    Write_Amplitude(0, gFrequencyCmd.amplitude);
    // ...
}
```
