## 数据包格式

```
55 02 00 00 00 03 05 FF
├─ 55       帧头（固定）
├─ 02       命令符（0x02 = 设置幅值）
├─ 00 00 00 03   幅值数据（大端序，十进制3表示3V）
├─ 05       校验码（前面所有数据的累加和）
└─ FF       帧尾（固定）
```

---

## 线性转换关系

### 已知条件
- `Write_Amplitude(0, 1000);` → 输出≈270mV
- `Write_Amplitude(0, 2000);` → 输出≈540mV

### 转换公式

```
Write_Amplitude参数 = (输入电压V * 1000000) / 270
```

### 常见设置值

| 目标电压 | 串口数据 | Write_Amplitude参数 | 数据包 |
|--------|--------|------------------|--------|
| 1V | 00 00 00 01 | 3703 | `55 02 00 00 00 01 02 FF` |
| 2V | 00 00 00 02 | 7407 | `55 02 00 00 00 02 03 FF` |
| 3V | 00 00 00 03 | 11111 | `55 02 00 00 00 03 05 FF` |
| 4V | 00 00 00 04 | 14815 | `55 02 00 00 00 04 06 FF` |
| 5V | 00 00 00 05 | 18518 | `55 02 00 00 00 05 07 FF` |

### 计算示例

要设置2.5V的幅值：
```
目标电压：2.5V
Write_Amplitude参数 = (2.5 * 1000000) / 270 = 9259
转换为16进制：0x2438
大端序表示4字节：00 00 24 38
校验码 = 0x02 + 0x00 + 0x00 + 0x24 + 0x38 = 0x5E
数据包：55 02 00 00 24 38 5E FF
```

## 代码集成说明

### 在 main.c 中自动完成的操作：

1. ✅ 初始化阶段
   ```c
   ZPN_UART_Init();           // 启动UART2接收
   FrequencyControl_Init();   // 注册PACK模板（必须在前）
   AmplitudeControl_Init();   // 共享模板
   ```

2. ✅ 主循环处理
   ```c
   while(1) {
       FrequencyControl_Process();  // 处理频率命令
       AmplitudeControl_Process();  // 处理幅值命令
   }
   ```

### 架构设计

- **独立模块**：frequency_control.c 和 amplitude_control.c 各自独立
- **共享框架**：两个模块共享UART2和PACK解析框架
- **互不干扰**：根据cmd字段（0x01=频率，0x02=幅值）自动区分
- **参数独立**：修改频率不会影响幅值，反之亦然

## 常见问题

**Q: 我同时改变频率和幅值，会互相影响吗？**  
A: 不会。两个模块独立处理，只修改各自对应的参数。

**Q: 线性转换的系数从哪来？**  
A: 根据注释中的实测值：1000参数→270mV。线性关系确保了任意电压都能正确转换。

**Q: 如何计算校验码？**  
A: 校验码 = (cmd + byte[0] + byte[1] + byte[2] + byte[3]) & 0xFF

例如，幅值3V的数据包：
```
cmd=0x02, data=0x00000003
checksum = (0x02 + 0x00 + 0x00 + 0x00 + 0x03) & 0xFF = 0x05 ✓
```
