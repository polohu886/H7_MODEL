# 频率控制模块集成快速指南

## 概述

本指南说明如何集成串口屏频率控制模块到现有项目。该模块允许通过UART2从串口屏实时改变AD9959的输出频率。

---

## 快速开始（3步）

### 1️⃣ 添加头文件（main.c）

在 `main.c` 的 `#include` 部分添加：

```c
#include "../MY_DSP/MY_Uart/frequency_control.h"
```

### 2️⃣ 初始化模块（main函数）

在 `main()` 中的 `AD9959_IO_Update()` 后添加：

```c
/* Initialize frequency control module to handle UART commands */
FrequencyControl_Init();
```

### 3️⃣ 主循环中处理命令（while循环）

在 `while(1)` 循环内添加：

```c
while (1)
{
    /* Process frequency control commands from UART */
    FrequencyControl_Process();
}
```

**完成！** 现在单片机可以接收来自串口屏的频率设置命令了。

```
串口屏发送格式：
┌─────┬──────┬──────┬──────┬──────┬──────┬────────┬────────┐

│0x55 │0x01  │ H    │ M-H  │ M-L  │ L    │  Sum   │ 0xFF   │
├─────┼──────┼──────┼──────┼──────┼──────┼────────┼────────┤
│ 1B  │ 1B   │ 1B   │ 1B   │ 1B   │ 1B   │  1B    │  1B    │
└─────┴──────┴──────┴──────┴──────┴──────┴────────┴────────┘

例子：设置1000 Hz
55 01 00 00 03 E8 EC FF
┌─────┬──────┬──────┬──────┬──────┬──────┬────────┬────────┐

       ↑      ↑───── 0x000003E8 = 1000 (十进制) ────↑

校验码计算：0x01 + 0x00 + 0x00 + 0x03 + 0xE8 = 0xEC
```

## 工作原理流程图

```
┌────────────────────────────────────────────────────────────┐
│ 串口屏通过UART2发送数据包                                    │
│ 例：55 01 00 00 03 E8 EC FF (设置频率1000Hz)              │
└────────────────────┬─────────────────────────────────────┘
                     ↓
┌────────────────────────────────────────────────────────────┐
│ UART2中断处理：数据写入环形缓冲区                             │
│ (由ZPN_Uart.c自动处理，无需改动)                            │
└────────────────────┬─────────────────────────────────────┘
                     ↓
┌────────────────────────────────────────────────────────────┐
│ main loop周期调用: FrequencyControl_Process()              │
└────────────────────┬─────────────────────────────────────┘
                     ↓
┌────────────────────────────────────────────────────────────┐
│ 1. PACK_ParseFromRingBuffer()                              │
│    └─ 从环形缓冲读取数据，查找帧头(0x55)和帧尾(0xFF)      │
└────────────────────┬─────────────────────────────────────┘
                     ↓
┌────────────────────────────────────────────────────────────┐
│ 2. PACK_ParseFrame()                                        │
│    ├─ 验证帧长度和帧头/帧尾                                │
│    ├─ 计算校验码：0x01 + 0x00 + 0x00 + 0x03 + 0xE8 = 0xEC│
│    ├─ 与接收的0xEC对比，通过则继续，否则丢弃              │
│    └─ 解析数据：cmd=0x01, frequency=0x000003E8=1000      │
│       (自动填充到全局结构体 gFrequencyCmd)                │
└────────────────────┬─────────────────────────────────────┘
                     ↓
┌────────────────────────────────────────────────────────────┐
│ 3. 检查命令: if (cmd == 0x01)                              │
└────────────────────┬─────────────────────────────────────┘
                     ↓
┌────────────────────────────────────────────────────────────┐
│ 4. 调用AD9959驱动更新频率：                                │
│    Write_Frequence(0, 1000);                               │
│    Write_Amplitude(0, 500);                                │
│    Write_Phase(0, 0);                                      │
```

## 校验码计算算法

```c
// 伪代码
checksum = (cmd + freq_byte[0] + freq_byte[1] + freq_byte[2] + freq_byte[3]) & 0xFF

// 例子：1000 = 0x000003E8
cmd = 0x01
freq_bytes = [0x00, 0x00, 0x03, 0xE8]
checksum = (0x01 + 0x00 + 0x00 + 0x03 + 0xE8) & 0xFF
         = 0x00EC & 0xFF
         = 0xEC ✓
```

## 常见问题（FAQ）

**Q1: 如何改变频率更新的通道？**
> 在 `frequency_control.c` 的 `Write_Frequence(0, frequency);` 中，第一个参数是通道号。改为 `Write_Frequence(1, frequency);` 即可改为通道1。

**Q2: 如果串口屏发送的频率超出AD9959的工作范围怎么办？**
> 可以在 `FrequencyControl_Process()` 中添加范围检查：
> ```c
> if (gFrequencyCmd.frequency > MAX_FREQUENCY || gFrequencyCmd.frequency < MIN_FREQUENCY) {
>     // 丢弃或提示错误
>     return;
> }
> ```

**Q3: 能否支持多个UART通道同时接收频率命令？**
> 可以！复制 `frequency_control.c` 的逻辑，为UART1创建 `FrequencyControl_Process_UART1()`，在main loop中并行调用。

**Q4: 校验码算法为什么用累加和？**
> 简单的累加和可以快速检测传输错误。如需更强的纠错能力，可改用CRC-8或CRC-16。
