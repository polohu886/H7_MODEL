# 幅值控制 - 故障诊断与修复

## 问题所在

**原始架构缺陷**：两个独立的全局变量，但只有一个被PACK框架填充。

```c
// 错误：PACK只注册和填充gFrequencyCmd
FrequencyControl_Init() → PACK_SetTemplate(gFrequencyCmd)

// 错误：但幅值模块检查自己的变量，永远为0
AmplitudeControl_Process() 
  if (gAmplitudeCmd.cmd == 0x02)  ← gAmplitudeCmd永远是{0}，不会被PACK填充！
```

---

## 修复方案

### 改为共享全局变量

```c
// frequency_control.c
FrequencyCommand_t gFrequencyCmd = {0};  // ← 改为public（去掉static）
PACK_SetTemplate(&gFrequencyCmd);        // ← 只注册一个

// frequency_control.h
extern FrequencyCommand_t gFrequencyCmd; // ← 导出供其他模块使用

// amplitude_control.c
#include "frequency_control.h"            // ← 导入全局变量
void AmplitudeControl_Process() {
    PACK_ParseFromRingBuffer();
    if (gFrequencyCmd.cmd == 0x02) {     // ← 检查同一个变量！
        // gFrequencyCmd.frequency 这里存的是幅值数据
    }
}
```

### 关键改动

| 文件 | 改动 | 原因 |
|------|------|------|
| frequency_control.c | `static` → 移除 | 使gFrequencyCmd全局可访问 |
| frequency_control.h | 添加`extern` | 其他模块可导入访问 |
| amplitude_control.c | 导入frequency_control.h | 访问gFrequencyCmd |
| amplitude_control.c | 移除gAmplitudeCmd | 改用共享的gFrequencyCmd |
| amplitude_control.c | 检查gFrequencyCmd.cmd | 不再检查gAmplitudeCmd.cmd |

---

## 现在的执行流程

```
串口屏发送幅值命令：55 02 00 00 00 03 05 FF
    ↓
UART2接收 → 环形缓冲区
    ↓
PACK_ParseFromRingBuffer()
    ↓
PACK_ParseFrame() ← 填充gFrequencyCmd
    ├─ gFrequencyCmd.cmd = 0x02（幅值命令）
    └─ gFrequencyCmd.frequency = 3（幅值数据）
    ↓
FrequencyControl_Process() 检查 (gFrequencyCmd.cmd == 0x01)
    ├─ 不匹配，直接return
    ↓
AmplitudeControl_Process() 检查 (gFrequencyCmd.cmd == 0x02)
    ├─ ✓ 匹配！
    ├─ write_param = (3 * 1000000) / 270 = 11111
    ├─ Write_Amplitude(0, 11111)
    ├─ AD9959_IO_Update()
    ↓
✓ AD9959输出幅值为3V
```

## 架构总结

**现在的架构**：

```
┌─────────────────────────────────────┐
│       UART2 接收中断                  │
│    ZPN_UART_Init()                   │
└──────────────┬──────────────────────┘
               ↓
        ┌──────┴──────┐
        ↓             ↓
    ┌─────────┐  ┌──────────┐
    │PACK框架 │  │环形缓冲区 │
    └────┬────┘  └──────┬───┘
         ↓              ↓
        gFrequencyCmd（通用）
             ↓
        ┌────┴────┐
        ↓         ↓
    Frequency  Amplitude
    (cmd=0x01) (cmd=0x02)
        │         │
        ↓         ↓
    Write_Frequence  Write_Amplitude
        │         │
        └────┬────┘
             ↓
        AD9959_IO_Update()
```

---

## 为什么之前不行？

1. ❌ PACK_SetTemplate() 只注册了 `gFrequencyCmd`
2. ❌ amplitude_control.c 创建了 `gAmplitudeCmd`，但它永远得不到PACK_ParseFrame的填充
3. ❌ `if (gAmplitudeCmd.cmd == 0x02)` 每次都false，因为gAmplitudeCmd永远是{0,0}

现在已全部修复！
