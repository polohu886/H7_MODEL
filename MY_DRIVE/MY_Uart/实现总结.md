## 🚀 工作原理

### 系统数据流

```
[串口屏] ─── UART2 ──→ [UART2 RX Interrupt] ─→ [Ring Buffer]
                                                     ↓
[main loop] ─→ FrequencyControl_Process()
                     ↓
           PACK_ParseFromRingBuffer()
                     ↓
           [查找帧头0x55 & 帧尾0xFF]
                     ↓
           [验证校验码] ✓/✗
                     ↓
           [解析4字节频率数据 (大端序)]
                     ↓
           [调用Write_Frequence()]
                     ↓
           [调用AD9959_IO_Update()]
                     ↓
           [✓ AD9959输出频率已更新]
```

### 串口数据包格式

```
帧结构：[帧头] [命令] [频率-3] [频率-2] [频率-1] [频率-0] [校验] [帧尾]
值示例：[0x55] [0x01] [0x00]  [0x00]  [0x03]  [0xE8]  [0xEC] [0xFF]

字段说明：
- 帧头(0x55)：标识数据包开始
- 命令(0x01)：0x01 = 设置频率命令
- 频率：4字节大端序整数
  例：0x000003E8 = 1000Hz
- 校验码：所有数据字节的累加和 & 0xFF
  计算：(0x01 + 0x00 + 0x00 + 0x03 + 0xE8) & 0xFF = 0xEC
- 帧尾(0xFF)：标识数据包结束
```

### 校验码计算

```c
checksum = (cmd + freq_byte[0] + freq_byte[1] + freq_byte[2] + freq_byte[3]) & 0xFF

例：频率1000 = 0x000003E8
checksum = (0x01 + 0x00 + 0x00 + 0x03 + 0xE8) & 0xFF
         = 0xEC
```

### 2. 使用Python工具验证校验码
```bash

# 生成特定频率的正确数据包

```

### 3. 逐步排查
```
现象：频率不更新
↓
1. 检查UART2是否接收到数据
   → 在中断处理中添加LED闪烁或printf
   
2. 检查数据包格式是否正确
   → 用串口监视器观察原始十六进制
   
3. 检查校验码是否正确
   → 使用packet_generator.py验证
   
4. 检查Write_Frequence()是否被调用
   → 在函数内添加printf或示波器观测
```
