# 9959_scan 扫频驱动使用说明

## 功能

控制 AD9959 DDS 芯片在指定频率范围内进行扫频输出，每步同时设置频率和幅值。

## 前置条件

- `Init_AD9959()` 已调用（初始化 AD9959 PLL 和默认参数）
- `Delay_Init()` 已调用（DWT 延时初始化）

## API

```c
void SCAN_RunSweep(uint32_t start_hz, uint32_t stop_hz, uint32_t step_hz,
                   uint16_t amplitude, uint32_t settle_ms);
```

| 参数 | 说明 |
|------|------|
| `start_hz` | 起始频率 (Hz) |
| `stop_hz` | 结束频率 (Hz) |
| `step_hz` | 频率步进 (Hz)，不能为 0 |
| `amplitude` | 幅值 (0-1023)，0 最小，1023 最大 |
| `settle_ms` | 每步停留时间 (ms) |

## 使用示例

```c
#include "9959_scan.h"
#include "ad9959.h"
#include "delay.h"

int main(void)
{
    // ... 系统初始化 ...
    Delay_Init();
    Init_AD9959();

    // 1kHz → 500kHz 扫频，步进 500Hz，幅值 1023，每步 100ms
    SCAN_RunSweep(1000, 500000, 500, 1023, 100);
    // 扫频结束后停在 500kHz

    while (1);
}
```

## 注意事项

1. **扫频是阻塞调用**，完成前不会返回。总耗时 ≈ `(stop - start) / step × settle_ms`
2. **幅值范围 0-1023**，对应 AD9959 的 10 位幅度控制字
3. **步进不能为 0**，否则函数直接返回
4. **stop 必须大于 start**，否则直接返回
5. 当前仅支持 **通道 0**（CH0），如需其他通道需修改 `9959_scan.h` 中的 `SWEEP_CHANNEL`
6. 扫频结束后频率停留在最后一次设置的值（`stop_hz`），幅值也保持在设定值
7. 依赖 `Write_Frequence()` 和 `Write_Amplitude()`，这两个函数的频率计算公式基于 500MHz 系统时钟

## CMake 工程配置

```cmake
# 添加驱动源文件
target_sources(${CMAKE_PROJECT_NAME} PRIVATE
    MY_DRIVE/MY_DDS_FUN/9959_scan.c
)

# 添加驱动头文件路径
target_include_directories(${CMAKE_PROJECT_NAME} PRIVATE
    MY_DRIVE/MY_DDS_FUN
)
```
